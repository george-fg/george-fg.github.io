Phase 0 Week 5 Technical Blog Part 5

Problem Solving

My response to a recent problem.

What was the problem?
Earlier in the course, I was struggling with CSS to style grid formations on my website. It was becoming a huge tizzy for me. I didn’t know where to start, and I could feel myself getting upset.

Problem solving techniques I used:
I initially did some grounding (mindfulness) to calm myself down. After this, I looked at some of my fellow cohort’s websites through developer tools. I also used Google extensively. With trial and error, I eventually got the ball rolling to the point where I was starting to enjoy myself.

How I felt throughout the process:
Stressed at the start – I was comparing myself to others, and actively putting myself down. Saying, “I cannot do this, I don’t have the capacity,” etc.
When I started to ground, I was able to calm my body. This helped me think clearly and move forward. The rest came eventually, kind of like a snowball. It started slowly gathering speed, until my mind was whooshing along.

What I learned:
I should try to watch my anxiety levels more at physiological level. I also need to watch my thoughts, and actively note them when they come up during the task. I also need to continue using the Pomodoro technique, as it has proved immensely helpful over the last couple of weeks.

Reflect on how confident you feel using problem solving techniques and processes for programming.
Pseudocoding – I have not had to use this yet, though it is definitely in the back of my mind as a tool to use. Maybe later this week?

Trying something – yep, definitely done this a few times. Trial and error. More often than not I get an error message back, but for something I think is really close; it works well.

Rubber ducky method – I haven’t done this with a rubber ducky per say, but I have certainly practiced speaking aloud. I kind of do this automatically. The other thing I have been doing is explaining some concepts to my girlfriend, which can also help. Speaking aloud helps to clarify what I am thinking.

Reading error messages – done, although understanding these error messages is difficult at times.
Console.logging – done for some Codecademy exercises, always helpful to see your output.

Googling – lots of this! This is my go-to method for problem solving on broad topics.

Asking your peers for help – I have asked my peers a fair bit for help so far. We have some great people in our cohort, which makes getting good answers really easy.

Asking coaches for help – I haven’t had to tag a coach yet. Maybe soon though.

Improving your process with reflection – I need to do more of this, especially after a difficult task. Sometimes I just want something to be over, so I end up rushing through it. Some reflection could slow me down and consolidate my learning.

Explain what the following functions do.

.map()
The .map() function allows you to take an array and copy it, but new characteristics.
For example, the following code takes an array of numbers and creates a new array containing the square roots of the numbers in the first array.

var numbers = [1, 4, 9]; var roots = numbers.map(Math.sqrt); // roots is now [1, 2, 3], numbers is still [1, 4, 9]

.filter()
The .filter() function allows you to create a new array out of a previous one, but only with desired elements. It “filters” out elements that are unwanted. It does this when you put in certain values when the function is called. These values will either be the wanted or unwanted elements in the new array.
The following example uses filter() to create a filtered array that has all elements with values less than 10 removed.

function isBigEnough(value) {
return value >= 10;
}
var filtered = [12, 5, 8, 130, 44].filter(isBigEnough); // filtered is [12, 130, 44]

.reduce()
The .reduce() function will take an array and reduce it to a certain result, depending on the value(s) used at the time of calling.
