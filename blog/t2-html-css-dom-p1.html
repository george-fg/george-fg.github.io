How the DOM, HTML, CSS, and JavaScript work together

A good way of explaining this is through an analogy of the human body.

First and foremost we have HTML, which stands for HyperText Markup Language. HTML is a bit like the human skeleton, it is the basic framework on which a webpage is built. It always comes first.

Coming up next is JavaScript, which essentially programs the behavior of a webpage and allows us to do all the cool stuff, like displaying a clock with an updating time. In that way, it is similar to a brain and the organs of a human body.

And to complete our trio, we have CSS, which stands for Cascading Style Sheets. CSS is like human skin, and the clothing we wear. It is the superficial layer of a webpage, where we aim to make stuff look good through visual styling.

Bringing this all together, we get the Document Object Model, or DOM for short. The DOM is created when a web browser pulls together the information from the trio above, and displays that information on a webpage. In this way, the DOM is like a person, complete and complex.

Boxifying Design and the Box Model

Boxifying design is a way in which a web developer can think about elements on his or her webpage. All elements in HTML are essentially boxes - understanding this can make development a lot easier because square or rectangular boxes are simpler to adjust and plan for. The box model is the entirety of an element. The model includes properties such as padding, margins, and actual content.
